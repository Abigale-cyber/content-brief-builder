# Content Brief Builder

## 触发条件

- 用户说"整理 brief"、"先别写，先整理方向"
- 用户说"这个题怎么写"、"先定个方向"
- 用户提供了 topic-pool.md 中的某个选题，想展开
- 用户已有明确选题但还没想清楚怎么写

**不触发：**
- "帮我找选题" / "抓热点" → content-topic-radar
- "我有想法但说不清楚" → interview-to-draft
- "写文章" / "帮我写公众号" → wechat-article-writer

## 不做什么

- 不写正文
- 不设计标题方向、开头草稿、结尾草稿
- 不做选题发现
- 不做包装改稿
- 不做发布

## 执行流程

### Step 1: 接收选题

从以下来源接收：
- topic-pool.md 中的某个选题条目
- 用户直接输入的一句话选题

如果信息不足，一次性确认（不逐个追问）：
1. 你主要想说什么？
2. 写给谁看？
3. 希望读者看完做什么？

### Step 2: 补全素材

读取 `references/research-methods.md`：
1. 先从用户提供的信息中提炼
2. 不够时联网搜索（可选，不阻塞）
3. 仍不够时知识推断兜底

每条素材必须通过 相关性/具体性/可验证/冲突感 四项检查。

### Step 3: 锁定方向

**角度判定**（读取 `references/brief-quality-guide.md`）：
- 角度必须是一句话、有立场、与众不同的
- 听起来像维基百科 = 不合格

**框架选择**：

| 框架 | 适用场景 | 结构 |
|------|---------|------|
| 故事类 | 传递价值观、建立人设 | 事件 → 冲突 → 转折 → 感悟 |
| 观点类 | 输出立场、引发讨论 | 观点 → 论据 → 总结 |
| 热点类 | 蹭热点、带观点 | 热点引入 → 分析 → 关联自身 |
| 清单类 | 干货分享、步骤教学 | 总述 → 分点列举 → 总结 |
| 教学类 | 教程、方法论 | 问题 → 方法 → 步骤 → 案例 → 总结 |

**SCQA 结构**：
- 情境(S)：读者熟悉的场景
- 冲突(C)：场景中的矛盾
- 问题(Q)：基于冲突的核心问题
- 答案(A)：你的解决方案或观点

### Step 4: 输出 Brief

读取 `../_shared/intermediate-format.md` 中 content-brief.md 的格式规范，按格式输出。

输出到本地文件：`content-brief-<选题关键词>.md`

### Step 5: 质量自检

读取 `references/brief-quality-guide.md` 中的 6 项自检：
1. 角度够硬？
2. 读者够具体？
3. 素材够硬？（≥3条）
4. 框架选对？
5. 有风险意识？
6. SCQA 完整？

任何一项不通过，回去改。

### Step 6: 用户确认

展示 brief 摘要，确认：核心判断、切入角度、框架。
有修改意见则更新后重新确认。

## Guardrails

- 角度必须有明确立场，不接受"可以从多个角度分析"
- 读者必须具体，不接受"所有人"
- 素材至少 3 条，不够就补（联网或推断），在 brief 中标注来源可信度
- 必须包含 SCQA 四个部分
- 必须包含风险提醒
- 用户说"直接写"时提醒先确认 brief
- 联网搜索失败时用知识推断兜底，不阻塞输出
