# Brief 质量标准

## 最关键的字段：切入角度

角度决定这篇文章跟别人有什么不同。

### 好角度

- 一句话说清楚
- 有明确立场（不是"这件事有两面性"）
- 跟已有内容不一样
- 有数据或经验支撑

好角度示例：
- "大多数人用 AI 做内容失败，不是因为工具不行，是因为没有质量文化"
- "公众号还值得做，但做法要变——从写文章变成搭流水线"

### 烂角度

- "AI 工具的全面介绍"（没有立场）
- "关于XX你需要知道的一切"（维基百科不是角度）
- "XX的最佳实践"（谁都能写的废话）

### 检验方法

两个不同写手拿到这份 brief，能写出方向一致的文章吗？不能 = 角度不够具体。

## Brief 反模式

| 反模式 | 问题 | 修复 |
|--------|------|------|
| "写一篇关于XX的全面指南" | 没有角度 | 定义具体立场 |
| 缺少读者定义 | 写手猜读者 | 写明具体读者画像 |
| 没有素材来源 | 写手编事实 | 至少 3 条可追溯素材 |
| 角度像维基百科 | 不是角度 | 找独特的、有争议的视角 |
| "所有人都应该看" | 读者太泛 | 缩窄到某一类人 |
| 复述热点标题 | 没有自己的角度 | 翻译成"这对读者有什么用" |
| 没有风险提醒 | 不知道哪里翻车 | 至少 1 个潜在风险 |

## 6 项自检（输出前必须全过）

1. [ ] 角度够硬？（一句话、有立场、与众不同）
2. [ ] 读者够具体？（不是"所有人"）
3. [ ] 素材够硬？（至少 3 条，每条有来源）
4. [ ] 框架选对？（和内容类型匹配）
5. [ ] 有风险意识？（至少 1 个翻车点）
6. [ ] SCQA 完整？（情境、冲突、问题、答案都有内容）

任何一项不通过，回去修改后再输出。
